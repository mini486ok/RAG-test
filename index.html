    상세 설정

    🔌 Ollama 서버에 연결할 수 없습니다

    이 페이지는 브라우저에서 직접 로컬 Ollama에 접속합니다. 아래를 확인해 주세요.

    1. Ollama 실행 확인 — 터미널에서 ollama list가 동작해야 합니다.
    2. CORS 허용 설정 — 웹페이지(https)에서 접속하려면 환경변수 설정이 필요합니다.

      Windows (PowerShell — 설정 후 Ollama 재시작)

      setx OLLAMA_ORIGINS "https://mini486ok.github.io"

      macOS

      launchctl setenv OLLAMA_ORIGINS "https://mini486ok.github.io"

      Linux (systemd)

      sudo systemctl edit ollama.service
# [Service] 섹션에 추가:
Environment="OLLAMA_ORIGINS=https://mini486ok.github.io"

      특정 오리진만 허용하는 위 방식을 권장합니다. "*"는 모든 웹사이트가 로컬 Ollama에 접근할 수 있게 되므로 편의용으로만 쓰세요. 설정 후 Ollama를 완전히 종료했다가 다시 실행해야 적용됩니다. (Windows: 트레이 아이콘 우클릭 → Quit)

    3. 브라우저 — 로컬 네트워크 접근 허용 (가장 흔한 원인) — Chrome은 https 페이지가 localhost에 접속할 때 권한을 요구합니다.
      • "로컬 네트워크의 기기를 찾고 연결" 알림이 뜨면 허용을 선택하세요.
      • 알림 없이 계속 실패하면(이전에 차단됨): 주소창 왼쪽 아이콘(🔒 또는 ⚙) 클릭 → 사이트 설정 → "로컬 네트워크 기기 액세스"를 허용으로 바꾸고 새로고침하세요.
    4. 대안 — 위 설정이 번거로우면 ① 저장소를 내려받아 로컬(start.bat)에서 실행하거나, ② 인증 프록시+터널 공유 링크(?server=…)로 접속하면 이 권한 설정 없이 동작합니다.

    연결 없이도 페이지 탐색은 가능하지만, 질의 실행과 DB 구축은 Ollama 연결이 필요합니다.
